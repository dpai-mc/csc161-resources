🖥️ VM Creation Workflow

Step-by-step process for creating a virtual machine:

  1. Navigate to node → Create VM
  2. Configure general settings (VM ID, name)
  3. Select operating system and ISO
  4. Configure virtual hardware
  5. Set up network configuration
  6. Review and create

🔄 System Update Process

Safe system update procedure:

  1. Check current system status
  2. Review available updates
  3. Apply updates systematically
  4. Reboot if needed
  5. Verify system after updates

🧠 Knowledge Check Quiz

Question 1: Navigation and Management

Where would you configure cluster-wide backup jobs in ProxMox VE?

Node → Backup
Datacenter → Backup
Storage → Backup
VM → Backup
✅ Correct! Datacenter → Backup is where you configure cluster-wide backup jobs that affect multiple nodes and VMs.
❌ Incorrect. Cluster-wide backup jobs are configured at the Datacenter level, not at individual node or VM levels.

Question 2: Console Access

Which console type provides the best web browser compatibility without requiring additional plugins?

SPICE
noVNC
xterm.js
VNC Viewer
✅ Correct! noVNC is HTML5-based and runs entirely in web browsers without requiring additional plugins or software.
❌ Incorrect. While other options may offer different features, noVNC provides the best browser compatibility without plugins.

Question 3: Storage Management

What type of content is stored in a storage backend configured for "Disk image" content type?

ISO images and VM templates
Virtual machine hard disk files
Backup files and snapshots
Container templates
✅ Correct! "Disk image" content type stores virtual machine hard disk files in formats like .qcow2, .raw, and .vmdk.
❌ Incorrect. "Disk image" content type specifically refers to VM disk files, not ISO images, backups, or templates.

Question 4: System Updates

What should you do FIRST before applying system updates in ProxMox VE?

Immediately click "Upgrade"
Click "Refresh" to update package lists
Reboot the system
Shutdown all VMs
✅ Correct! Always refresh the package lists first to ensure you see the latest available updates before applying them.
❌ Incorrect. You should always refresh the package lists first to get the most current update information.

Question 5: Interface Navigation

Which section would you use to view real-time CPU and memory usage graphs for a ProxMox node?

Node → Summary
Node → System
Datacenter → Summary
Node → Shell
✅ Correct! The Node → Summary page displays real-time graphs and metrics for CPU, memory, storage, and network usage.
❌ Incorrect. While other sections provide system information, the Summary page specifically shows the real-time usage graphs.
