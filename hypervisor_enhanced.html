üåü What is a Hypervisor?

Definition

A hypervisor, also known as a virtual machine monitor (VMM), is a crucial software layer in virtualization technology that enables multiple virtual machines (VMs) to run on a single physical machine or host. Think of it as the supervisor in charge of dispersing the components that make up VMs.

üîë Key Functions

  • Managing and allocating hardware resources like CPU, memory, and storage
  • Creating a virtualization layer that runs between the OS and server hardware
  • Allowing each VM to operate as if it were on a separate, dedicated machine isolated from other VMs
  • Scheduling operations so VMs don't overrun one another when using resources

üîÑ Types of Hypervisors

üéØ Type 1 - Bare Metal Hypervisor

A type 1 hypervisor, or a bare metal hypervisor, interacts directly with the underlying machine hardware. The type 1 hypervisor sits on top of the bare metal server and has direct access to the hardware resources.

Characteristics:

  • Runs directly on the underlying host system
  • Does not require any base server operating system
  • Most common in an enterprise data center or other server-based environments
  • Very efficient because they have direct access to physical hardware resources

Examples:

  • VMware ESXi
  • Microsoft Hyper-V
  • KVM (Kernel-based Virtual Machine)
  • Citrix XenServer

üñ•Ô∏è Type 2 - Hosted Hypervisor

The type 2 hypervisor is an application installed on the host operating system. It's also known as a hosted or embedded hypervisor.

Characteristics:

  • Run on a conventional operating system as a software layer or application
  • The type 2 hypervisor negotiates with the operating system to obtain underlying system resources
  • Better for individual users who want to run multiple operating systems on a personal computer
  • Easier to install, configure, and use than bare-metal hypervisors

Examples:

  • VMware Workstation
  • Oracle VirtualBox
  • VMware Fusion
  • Parallels Desktop

‚ú® Benefits of Hypervisors

üí∞ Cost Efficiency

By consolidating multiple virtual machines (VMs) onto a single physical server, organizations can significantly reduce both hardware and maintenance costs. Hypervisors help enterprises make more efficient use of the underlying physical equipment.

üîß Resource Optimization

Organizations use virtualization software like hypervisors because the software helps them to use resources efficiently and reduce hardware investment. Virtualization has substantially increased server utilization rates.

üîí Isolation and Security

Hyper-V runs each virtual machine in its own isolated space, which means you can run more than one virtual machine on the same hardware at the same time. A VM provides an environment that is isolated from the rest of a system, so whatever is running inside a VM won't interfere with anything else running on the host hardware.

üåê Flexibility and Portability

A hypervisor abstracts the host's hardware from the operating software environment. IT administrators can configure, deploy, and manage software applications without being constrained to a specific hardware setup.

üöÄ Use Cases

  • Server Consolidation: Companies use hypervisors to consolidate multiple computers that perform different functions into a single server
  • Development and Testing: Reproduce different computing environments without having to buy or maintain all the hardware
  • Virtual Desktop Infrastructure (VDI): Use a centralized desktop strategy with VDI to increase business agility and data security
  • Private Cloud: Provide more flexible, on-demand IT services by moving to shared resources
  • Desktop Virtualization: Employees use desktop virtualization software to emulate a version of their workstation computing environment on the server

üíº Popular Type 1 Hypervisors

üîµ VMware ESXi

VMware ESXi stands out as a prominent and widely adopted Type I hypervisor, known for its robust virtualization capabilities and efficient resource management in enterprise environments. VMware led the market in developing innovative features such as memory overcommitment, vMotion, Storage vMotion, Fault Tolerance, and more.

Key Features:

  • Comprehensive suite of features for managing virtualized environments
  • vSphere management platform enables centralized control and monitoring of multiple ESXi hosts
  • VM Mobility - enables moving virtual machines between physical servers, manually or automatically
  • Available in both free and commercial editions

üü¶ Microsoft Hyper-V

Hyper-V is Microsoft's hypervisor designed for use on Windows systems. Hyper-V is a built-in component of all modern versions of Windows. Hyper-V is a type-1 hypervisor. When you install Hyper-V, the setup converts the original OS into something like a VM and puts the hypervisor below.

Key Features:

  • Built-in component of Windows Server environments
  • Live migration, replication of virtual machines, dynamic memory
  • Seamless integration with Microsoft Azure for hybrid cloud deployments
  • Disaster recovery with Hyper-V Replica and backup capabilities

üêß KVM (Kernel-based Virtual Machine)

KVM is built into Linux as an added functionality that makes it possible to convert the Linux kernel into a hypervisor. KVM was merged into the Linux kernel in 2007, so if you're using a modern version of Linux, you already have access to KVM.

Key Features:

  • Open-source solution with all the features of Linux
  • Live migration, scheduling and resource control, and higher prioritization
  • Can be added to most Linux operating systems, including Ubuntu, SUSE and Red Hat Enterprise Linux
  • Supports virtualization extensions that Intel and AMD built into their processor architectures

üí° Essential Terminology

Datastore: A logical storage container that holds virtual machine files, templates, and ISO images.
vSphere Client: The web-based management interface used to manage ESXi hosts and virtual machines.
VMkernel: The virtualization layer that provides core services like memory management, process scheduling, and I/O stack.
vMotion: Technology that allows live migration of running virtual machines from one ESXi host to another.
VMFS: Virtual Machine File System - VMware's clustered file system designed for storing virtual machine files.
